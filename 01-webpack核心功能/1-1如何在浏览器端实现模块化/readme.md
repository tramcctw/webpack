# 浏览器模块化

每依赖一个 js 文件就会去请求一个 js 文件

## 问题

- 效率问题，精细的模块划分带来了更多的 js 文件，更多的 js 文件带来了更多的请求，降低了访问效率
- 兼容性问题：浏览器目前仅支持 es6 模块化标准，并且还存在兼容性问题
- 工具问题：浏览器不支持 npm 下载第三方包

根本原因 node 端读取的是本地文件

devtime

- 模块越细越好
- 不考虑兼容性的问题
- 支持 npm 等包管理器
- 支持多种模块化标准

runtime

- 文件越少越好
- 文件体积越小越好
- 所有浏览器都兼容
- 能够解决执行效率问题

## 构建工具

将 devtime 代码转换为 runtime 代码

常见的构建工具
webpack
grunt
gulp
